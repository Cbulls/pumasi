# 품앗이폼 — 크레딧 계수 보정 A/B 실험 설계서 v1.0

## 0. 이 문서의 목적

크레딧 비용 계산식의 계수(주관식 가중, 지급률 등)는 출시 전까지 모두 추정값이다.
본 문서는 출시 직후 실데이터로 이 계수들을 보정하기 위한 실험을 설계한다.

대상 계산식 (상세기획서 B-1):
```
응답 1건 비용 = max(1, round(
    1 + estimated_min × W_time
      + essay_ratio × estimated_min × W_essay
      + question_count × W_count
))
응답자 적립 = round(비용 × payout_rate)
```

조정 대상 계수:
- `W_essay` (주관식 가중) — 근거가 가장 약함. 최우선 실험 대상.
- `payout_rate` (지급률, 1 − 소각률) — 통화 안정성 ↔ 응답 동기의 트레이드오프.
- `W_time` = 1.0 은 단위 앵커이므로 실험 제외 (바꿔도 리디노미네이션일 뿐).
- `W_count` = 0.05 는 영향이 미미하므로 1차 실험 제외, 추후 단독 검토.

## 1. 핵심 가설

| ID | 가설 | 귀무가설(H0) |
|---|---|---|
| H1 | 주관식 보상을 높이면 주관식 비중이 큰 설문의 완료율이 올라간다 | W_essay는 완료율에 영향 없다 |
| H2 | 지급률을 높이면 응답자의 응답 빈도(주간 응답 수)가 올라간다 | payout_rate는 응답 빈도에 영향 없다 |
| H3 | 지급률을 낮추면 크레딧 부족이 빨리 와 현금 구매 전환율이 올라간다 | payout_rate는 구매 전환율에 영향 없다 |

H2와 H3은 방향이 반대다. 지급률이 높으면 응답자는 행복하지만(H2) 크레딧이 안 마르므로 구매가 안 일어난다(H3).
이 둘의 균형점을 찾는 것이 지급률 실험의 본질이다.

## 2. 지표 정의

### 2-1. 주요 지표 (Primary)
| 실험 | 주요 지표 | 정의 |
|---|---|---|
| 실험 A (W_essay) | 설문 완료율 | 응답 시작 대비 정상 제출(quality_flag=pass) 비율 |
| 실험 B (payout_rate) | 주간 응답 수 / 활성 응답자 | 응답 동기의 직접 측정치 |

### 2-2. 보조 지표 (Secondary / Guardrail)
- 응답 중도 이탈률 (시작 후 미제출)
- 응답 1건당 평균 소요시간 (성의 측정 대용치)
- 불성실 응답 비율 (quality_flag = hold/reject 비율) ← **가드레일: 악화 시 실험 중단**
- 현금 구매 전환율 (활성 사용자 중 구매 발생 비율)
- 신규 설문 등록 수 (제작자 측 건강도)
- 7일 재방문율 (리텐션)

### 2-3. 가드레일 지표 (Guardrail)
아래가 통제군 대비 유의하게 악화되면 해당 변형군을 즉시 중단한다.
- 불성실 응답 비율 +5%p 이상 상승
- 7일 재방문율 −3%p 이상 하락

## 3. 실험 설계

### 3-1. 실험 A — 주관식 가중(W_essay)
- 무작위 배정 단위: **응답자(user)** 단위 (※ 배정 단위 주의사항은 6장 참고)
- 변형군:
  - 통제군 C: W_essay = 1.5 (현재 기본값)
  - 처치군 T1: W_essay = 1.0 (보상 약)
  - 처치군 T2: W_essay = 2.0 (보상 강)
- 트래픽 분할: C / T1 / T2 = 34% / 33% / 33%
- 관찰 집단: 주관식 비율이 일정 이상(예: essay_ratio ≥ 0.2)인 설문에 대한 응답만 분석에 포함
  (주관식이 거의 없는 설문은 W_essay 변화의 영향을 받지 않으므로 노이즈)

### 3-2. 실험 B — 지급률(payout_rate)
- 무작위 배정 단위: **응답자(user)** 단위
- 변형군:
  - 통제군 C: payout_rate = 0.80 (소각 20%)
  - 처치군 T1: payout_rate = 0.85 (소각 15%)
  - 처치군 T2: payout_rate = 0.70 (소각 30%)
- 트래픽 분할: 34% / 33% / 33%
- 주요 관찰: T1은 H2(응답 동기↑) 검증, T2는 H3(구매 전환↑) 검증

### 3-3. 두 실험의 동시 진행
- 실험 A와 B는 **직교(orthogonal)** 하게 동시 배정 가능 (W_essay × payout_rate)
- 단, 표본이 충분할 때만. 초기 트래픽이 적으면 **순차 진행**(A 먼저, B 나중) 권장.
- 동시 진행 시 교호작용(interaction) 항을 분석 모델에 포함할 것.

## 4. 표본 크기 & 기간

### 4-1. 표본 크기 산정 (참고용 예시)
완료율(실험 A) 기준, 다음 가정으로 계산:
- 기준 완료율(baseline): 60% (가정 — 실제 파일럿 값으로 교체)
- 검출하려는 최소 효과크기(MDE): ±5%p
- 유의수준 α = 0.05 (양측), 검정력 1−β = 0.80

→ 비율 검정 표본 공식:
```
n ≈ (Z_α/2 + Z_β)² × [p1(1−p1) + p2(1−p2)] / (p1 − p2)²
  = (1.96 + 0.84)² × [0.60×0.40 + 0.55×0.45] / (0.05)²
  ≈ 7.84 × (0.24 + 0.2475) / 0.0025
  ≈ 7.84 × 0.4875 / 0.0025
  ≈ 약 1,529 (군당, 분석 대상 응답 건수)
```
즉 **변형군당 약 1,500건의 (주관식 설문) 응답**이 필요. 3개군이면 ~4,500건.

> ⚠️ baseline·MDE 가정이 결과를 좌우한다. 파일럿 1~2주로 실제 baseline을 먼저 확보한 뒤 재계산할 것.
> 표본 계산은 statsmodels 등으로 다시 정밀 산출 권장.

### 4-2. 실험 기간
- 최소 운영 기간: **2주** (요일 효과 — 평일/주말 응답 패턴 차이 — 흡수 위해 최소 1주 × 2)
- 표본이 4-1 목표에 도달하기 전에는 종료하지 않는다.
- 최대 기간: 4주 (그 이상은 외부 환경 변화로 오염 위험)

## 5. 분석 방법

### 5-1. 1차 분석
- 완료율/전환율 등 비율 지표: 이항 비율의 z-검정 또는 카이제곱
- 주간 응답 수 등 카운트 지표: 음이항 회귀 (과분산 대응)
- 소요시간 등 연속 지표: 분포가 치우치므로 중앙값 + Mann-Whitney U

### 5-2. 다중비교 보정
- 군이 3개(C/T1/T2)이고 지표가 여러 개이므로 다중비교 문제 발생
- 주요 지표에는 본페로니 또는 Holm 보정 적용
- 보조 지표는 탐색적(exploratory)으로만 해석, 의사결정 근거로 삼지 않음

### 5-3. 의사결정 규칙
| 결과 | 조치 |
|---|---|
| 처치군이 주요 지표에서 유의하게 우세 + 가드레일 정상 | 해당 계수를 새 기본값으로 채택 |
| 유의차 없음 | 기존 기본값 유지 (또는 운영 단순성 위해 보수적 값 선택) |
| 가드레일 악화 | 즉시 해당 군 중단, 채택 불가 |
| H2 vs H3 상충 | 응답 동기와 구매 전환을 가중합한 사업 목표(예: 장기 LTV)로 판단 |

## 6. 위협 요인 & 통제 (Validity Threats)

이 실험은 일반 A/B와 다른 고유 위험이 있다. 양면 시장 + 공유 통화이기 때문.

### 6-1. 간섭(Interference) / SUTVA 위반 — 가장 큰 위험
- 문제: 크레딧은 사용자 간 거래되는 공유 자원이다. 응답자 A(처치군)가 더 많이 응답하면 그가 만든 설문도 더 많이 노출되어, 통제군 사용자의 경험까지 바뀐다. → 군 간 독립성 깨짐.
- 완화책:
  - (a) 가능하면 **시간 분할(switchback) 설계** 고려 — 전체 시스템에 한 계수를 며칠, 다른 계수를 며칠 적용해 비교. 간섭에 강하나 시계열 혼입 주의.
  - (b) 또는 **클러스터 무작위화** — 사용자를 상호작용이 적은 그룹(예: 카테고리·지역)으로 나눠 군 배정.
  - (c) 최소한, 응답자 단위 배정 시 "처치군 사용자가 만든 설문은 같은 처치군에게만 노출"하는 폐쇄 루프 구성 검토.

### 6-2. 신규 사용자 가입 보너스 오염
- 신규 가입 보너스 크레딧이 지급률 실험(B)의 효과를 가린다 (크레딧이 안 말라서).
- 완화: 분석에서 가입 N일 이내 신규 유저를 분리하거나, 보너스 소진 이후 구간만 분석.

### 6-3. 학습 효과 / 노벨티
- 출시 직후 사용자는 시스템에 익숙하지 않아 행동이 불안정.
- 완화: 첫 3~7일을 워밍업 구간으로 분석에서 제외(burn-in).

### 6-4. 계수 변경의 소급 문제
- 실험 중 계수가 바뀌면 이미 예치된 설문의 가격과 신규 설문 가격이 달라져 혼란.
- 규칙: 계수는 **신규 설문/신규 응답에만 적용**, 진행 중 예치는 등록 시점 계수로 고정.

## 7. 실험 운영 체크리스트

- [ ] 계수를 코드 상수가 아닌 **원격 config(feature flag)** 로 분리
- [ ] 사용자별 실험군 배정을 결정론적 해시(user_id 기반)로 고정 — 재배정 방지
- [ ] credit_ledger에 적용된 계수 버전(coeff_version)을 함께 기록 — 사후 추적용
- [ ] 파일럿 1~2주로 baseline 확보 → 표본 크기 재계산
- [ ] 가드레일 지표 실시간 대시보드 + 자동 알림(임계 초과 시)
- [ ] 분석 코드/지표 정의를 실험 시작 **전에** 사전등록(pre-register) — p-hacking 방지
- [ ] 종료 후 결과를 기본 config에 반영, 변경 이력 남기기

## 8. 요약

| 항목 | 내용 |
|---|---|
| 1순위 실험 | 주관식 가중 W_essay (1.0 / 1.5 / 2.0) — 근거가 가장 약함 |
| 2순위 실험 | 지급률 payout_rate (0.70 / 0.80 / 0.85) — 동기 vs 수익 균형 |
| 핵심 지표 | 완료율(A), 주간 응답 수(B) |
| 가드레일 | 불성실 응답 비율, 재방문율 |
| 최대 난점 | 공유 통화로 인한 군 간 간섭(SUTVA 위반) → switchback/클러스터 설계 검토 |
| 선결 과제 | 파일럿으로 baseline 확보 후 표본 크기 정밀 재계산 |

> 본 설계의 모든 수치(트래픽 분할, MDE, 기간, baseline)는 출발점 예시다.
> 파일럿 데이터로 baseline을 확정한 뒤 표본 크기와 효과크기를 다시 계산하는 것이 첫 실행 단계다.
