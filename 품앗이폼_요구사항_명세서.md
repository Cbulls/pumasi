# 품앗이폼(PumasiForm) 요구사항 명세서 v1.0

## 0. 서비스 개요

- **서비스명(가칭):** 품앗이폼 (PumasiForm)
- **핵심 가치:** 구글폼·네이버폼 수준의 설문 제작 기능 + "남의 설문에 응답해야 내 설문 결과를 볼 수 있는" 상호 응답(품앗이) 구조
- **해결 문제:** 설문 제작은 쉽지만 응답자 모으기가 어렵다(졸업논문, 1인 사업자, 마케터). → 응답 크레딧을 매개로 응답자 풀을 자체 순환시킨다.
- **확정 방향(의사결정 완료):**
  - 품앗이 정산 엔진 = **하이브리드** (풀 방식 기본 + 1:1 매칭 추천 레이어)
  - 수익 모델 = **크레딧 현금 구매 포함** (시간 없는 유저 전환 + 매출)

## 2. ★ 핵심: 품앗이(상호 응답) 시스템 — 하이브리드

### 2-1. 크레딧(Credit) 메커니즘
- 화폐 단위 **크레딧** 도입
- **획득:** 남의 설문 1건 정상 응답 → +N 크레딧
- **소모:** 내 설문 노출/등록 예치, 내 설문 결과 열람
- **핵심 규칙:** "크레딧 잔액 = 결과 열람 권한"

### 2-2. 하이브리드 엔진 (확정)
- **기본 = 풀(Pool) 방식:** 내 설문 노출에 크레딧 예치 → 누구나 응답 가능 → 응답 1건당 차감·응답자 지급. 확장성 확보.
- **레이어 = 1:1 매칭 추천:** "내 설문에 응답해준 사람의 설문을 우선 추천" → 상호성 강화.
- 효과: 확장성(풀) + 서로서로(매칭) 동시 달성.

### 2-3. 크레딧 가중치 설계 (남용 방지 핵심)
- "1응답=1크레딧" 단순화 금지 → 짧은 설문만 양산되는 문제.
- 비용 산정 변수: 질문 수, 예상 소요시간, 주관식 비율.
- 예: 소요시간 1분당 1크레딧. 5분 설문 응답 1건 받으려면 5크레딧 예치 → 나도 남의 5분 설문 1건 응답해야 메움.
- 신규 가입 보너스 크레딧 → 콜드스타트(닭-달걀) 완화.

### 2-4. 어뷰징 방지 (필수)
- 최소 응답 시간 검증(비정상 고속 제출 차단)
- 불성실 응답 탐지(주관식 반복문자, 객관식 일괄 동일선택 straight-lining)
- 주의 검증 문항(Attention check) 삽입 옵션
- 신고 & 평판(신뢰도 점수) 시스템
- 중복 계정·자동화 차단(이메일/휴대폰 인증, IP·디바이스 핑거프린팅, CAPTCHA)
- 크레딧 거래/현금화 금지 정책 + 비정상 적립 모니터링

### 2-5. 응답자 UX
- 추천 피드(카테고리·소요시간·보상 정렬)
- 타깃팅 필터(연령·성별·직업·지역) → 조건 일치 설문만 노출
- 진행 상태바 + 남은 크레딧 실시간 표시
- "완료 시 +N 크레딧" 보상 미리보기
- 모바일 우선 설계

### 2-6. 결과 열람 게이트 로직
```
내 설문 결과 열람 요청
  → 크레딧 잔액 확인
     ├─ 충분: 결과 대시보드 전체 열람
     └─ 부족: ① 남의 설문 응답 유도(1순위) ② 크레딧 구매(2순위)
              + 차트 블러 미리보기로 전환 유도
```

### 2-7. 수익 모델 — 크레딧 현금 구매 (확정 포함)
- 응답할 시간이 없는 유저 대상 크레딧 패키지 직접 판매
- 게이트에서 "응답하기" 1순위 / "구매하기" 2순위 보조 동선
- 향후 확장: 프리미엄(고급 분석·무제한 응답·로고 제거) 구독

## 3. 결과 엑셀 다운로드
- 포맷: `.xlsx`(기본) + `.csv`
- 구조: 1행=1응답, 1열=1질문(+응답시각, 익명 ID)
- 다중선택: 쉼표 구분 / 옵션별 0·1 열 분해 선택
- 시트 분리: Raw Data + 질문별 집계(빈도·비율)
- 필터 적용 상태 그대로 내보내기
- 대용량: 서버 비동기 생성 + 다운로드 링크

## 4. 결과 그래프(시각화)
- 객관식·드롭다운 → 막대/원형
- 체크박스 → 가로 막대(중복 합산)
- 선형 배율·평점 → 평균 + 분포 히스토그램
- 날짜/시간 → 타임라인
- 주관식 → 응답 리스트 + (확장) 워드클라우드
- 실시간 대시보드: 총 응답·완료율·평균 소요시간
- 교차분석(Cross-tab, 확장)
- 요약/개별 응답 토글, 차트 PNG 내보내기

## 5. 공통/비기능 요구사항
- 인증: 소셜 로그인(구글·카카오·네이버) + 이메일. 1인 1계정 강제(크레딧·신뢰도 무결성).
- 모바일 반응형 필수, 응답 페이지 로딩 최적화
- 개인정보: 응답자 익명성, 민감정보 동의, 개인정보보호법 준수
- 확장성: 인기 설문 동시 응답 트래픽 대응

### 제안 기술 스택(참고)
- 프론트: Next.js(React) + TypeScript
- 백엔드: NestJS(Node) 또는 FastAPI(Python)
- DB: PostgreSQL(설문·응답·크레딧 트랜잭션) + Redis(피드·세션 캐시)
- 엑셀: SheetJS/openpyxl + 비동기 큐(BullMQ/Celery)
- 인프라: AWS/GCP, 파일 업로드 S3 호환 스토리지
- 결제: 국내 PG(토스페이먼츠/포트원 등) 연동

## 6. 화면 와이어프레임 (핵심 3종)

### ① 응답 피드 (응답자 메인)
- 상단: 보유 크레딧 배지 / 필터(소요시간·카테고리·내게 맞춤)
- 카드: 제목, 대상·문항 정보, ⏱소요시간, 조건 일치 배지, "완료 시 +N" 보상, [응답하기]
- 원칙: 보상 강조 / 소요시간·문항수 필수 표기 / 타깃 일치 배지

### ② 결과 게이트 (크레딧 부족)
- 잠금 메시지 + 필요/보유 크레딧
- 차트 블러 미리보기
- [남의 설문 응답하기](1순위) / [크레딧 구매](2순위)

### ③ 결과 대시보드 (게이트 통과)
- 메트릭(총 응답·완료율·평균 소요시간)
- 요약/개별 토글, [엑셀][CSV] 다운로드
- 질문 유형별 자동 차트 매핑

## 7. MVP 범위 (우선순위)
- **1차 MVP:** 폼 빌더(핵심 8유형) + 응답 페이지 + 하이브리드 크레딧 엔진(풀 기반) + 기본 어뷰징 방지(최소 응답시간·인증) + 결과 대시보드(기본 차트) + 엑셀 다운로드 + 크레딧 구매(기본 PG 연동)
- **2차:** 1:1 매칭 추천 고도화, 타깃팅 필터, 불성실 응답 정밀 탐지, 평판 시스템, 교차분석, 프리미엄 구독
